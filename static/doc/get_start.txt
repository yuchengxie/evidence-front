## 简介

新生存证系统是依托于 [新生链（nb-chain）](http://www.nb-chain.net/) 的一套存证系统。新生链由深圳市红砖坊技术有限公司研发，拥有自主产权。

证据保全是区块链系统的基础功能，“新生存证” 产品尝试以规范的形式，提供通用的存证服务。它易于部署、易于移植，简单，易于上手，且具备广泛适应性。

本网站为 “新生存证” 产品提供一份详细的操作指导，我们开发了一套简易的 “存证前置机” 样例工程 evidence-front，新手对照样例源码，依序阅读这份材料，相信很快能入手搭建一套属于自己的 “存证前置机”。

存证前置机是一套存证辅助系统，通常运行在云计算中心，它用于汇总存证对象、代理存证操作、提供存证查询，并配套支持用户登录及相关权限管理。

&nbsp;

## 样例项目

本操作指导通过介绍 evidence-front 样例项目，引导大家快速入手存证系统的定制开发。

evidence-front 项目采用 python 开发，为简明起见，本项目未引入数据库，也未采用复杂组网，我们还尽力让样例代码变少、变精简，以便减轻学习负担。同时，本样例提供完整应用体验，代码能在实际环境正常运行。

因为存证需求通常不是孤立的，它要与具体业务配套使用，存证作为一项支撑功能，一般要求嵌入到其它业务中使用。所以，我们提供的样例实际是一种范本，大家可以照抄现有 Web 界面及服务侧设计，维持样例精简主要为这个目的服务。

&nbsp;

## 快速入门

evidence-front 要求在 python 3.4+ 上运行，下载源码后，在 shell 窗口进入源码工程目录。

我们先安装如下 python 依赖库，如已安装则跳过。

``` bash
  pip3 install flask
```

然后运行如下脚本，在本机启动一个 “存证前置机”。

``` bash
  python3 application.py
```

这个存证前置机将在本机的 3000 端口启动一个 Web 服务。然后在浏览器中访问 `localhost:3000`，显示界面如下：

![welcome](doc/pic/welcome.jpg)
